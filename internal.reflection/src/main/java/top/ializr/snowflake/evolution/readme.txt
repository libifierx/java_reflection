基于现有订单号格式AO+yyyymmdd+leftpad(seq,10,'0') . AO201708250000000010,AO201708250000000011..
这种订单格式暴露我们订单的数量，根据snowflake算法灵感，对我们订单号进行改造。


改造后格式如下：
	 前10长度格式不变AO+yyyymmdd
	 后13长度。
	 
	 
	 后13长度来源：（以下位代表bit）
	 
	 18bit  time
	 5bit 	data_center_id
	 5bit 	work_id
	 15bit  sequence
	 

	 time:我们取hhmmss部分,一天24*60*60=86400秒。需要17bit表达（2的16次幂65536,2的17次幂131072）。
	 data_center_id: 2的5次幂=32，能表达机房个数32个，[0,31]。最多32个机房
	 work_id：2的5次幂=32，能表达服务个数32个，[0,31]。单机房最多32个服务。
	 sequence: 2的15次幂32768，一秒钟最多可以生成32768个序列。一个服务一秒钟生成id个数。
